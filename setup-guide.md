# Claude Code でWebアプリを作る - セットアップ手順書

**対象者**: Claudeは使えるが、GitHubアカウントを持っていない方
**所要時間**: 約20分
**ゴール**: Claude Codeで会話しながらWebアプリを作り、公開できる状態にする

---

## 全体の流れ

```
Step 1  GitHubアカウントを作る（5分）
  ↓
Step 2  リポジトリを作る（3分）
  ↓
Step 3  Claude Codeを開く（1分）
  ↓
Step 4  GitHubと連携する（2分）
  ↓
Step 5  会話でアプリ作成を始める（すぐ）
  ↓
Step 6  GitHub Pagesで公開する（5分）
```

---

## Step 1: GitHubアカウントを作る

GitHubはソースコードを管理・公開するためのサービスです（無料）。

1. ブラウザで **https://github.com** を開く
2. 右上の **「Sign up」** をクリック
3. 以下を入力する
   - **Email**: 自分のメールアドレス
   - **Password**: パスワードを設定
   - **Username**: 英数字のユーザー名（例: `yamada-taro`）
4. メール認証を求められるので、届いたメールのリンクをクリック
5. プランは **「Free」** のままで **「Continue」**
6. アカウント作成完了

---

## Step 2: リポジトリを作る

リポジトリ = プロジェクトのフォルダのようなものです。

1. GitHubにログインした状態で、右上の **「+」** → **「New repository」** をクリック
2. 以下を設定する
   - **Repository name**: プロジェクト名を入力（例: `my-portal`）
   - **Public** を選択（無料でGitHub Pagesを使うため）
   - **「Add a README file」** にチェックを入れる
3. **「Create repository」** をクリック
4. リポジトリが作成される（URLは `https://github.com/ユーザー名/リポジトリ名`）

---

## Step 3: Claude Codeを開く

1. ブラウザで **https://claude.ai/code** を開く
2. Claudeのアカウントでログインする（普段使っているClaudeと同じアカウント）
3. Claude Codeの画面が表示される

> **補足**: Claude Codeは、通常のClaude（チャット）とは異なり、
> ファイルの作成・編集・Git操作ができる開発特化版です。

---

## Step 4: GitHubと連携する

Claude CodeからGitHubのリポジトリにアクセスできるようにします。

1. Claude Codeの画面で、セッション開始時に **GitHubとの連携** を求められる場合は画面の指示に従う
2. もしくは、新しいセッション作成時に **リポジトリを選択** する画面が表示される
3. Step 2で作成したリポジトリ（例: `my-portal`）を選択する
4. 連携が完了すると、リポジトリ内のファイルをClaude Codeから読み書きできるようになる

> **初回のみ**: GitHubへのアクセス許可を求められたら **「Authorize」** をクリックしてください。

---

## Step 5: 会話でアプリ作成を始める

ここからが本番です。日本語で作りたいものを伝えるだけです。

### 最初のメッセージ例

```
建設会社向けの社内ポータルサイトを作ってください。
お知らせ、業務アプリへのリンク、経営ダッシュボードを含めてください。
```

```
飲食店向けの予約管理ダッシュボードを作ってください。
今日の予約一覧、売上グラフ、空席状況が見れるようにしてください。
```

```
社内向けのFAQサイトを作ってください。
カテゴリ別に質問を整理して、検索機能もつけてください。
```

### Claude Codeがやってくれること
- HTMLファイル、CSS、JavaScriptを自動で作成
- ファイルをGitHubのリポジトリに保存（コミット＆プッシュ）
- 修正依頼にも会話で対応（「文字を大きくして」「色を変えて」など）

### コツ
- **具体的に伝える**: 「いい感じに」よりも「文字を大きめに」「青系の色で」のほうが望みの結果になりやすい
- **スクリーンショットを貼る**: 「この画面のここを直して」と画像付きで伝えるのが最も効果的
- **少しずつ進める**: 一度に全部作ろうとせず、まず基本を作ってから細かく修正していく

---

## Step 6: GitHub Pagesで公開する

作成したサイトをURLで誰でも見られるようにします（無料）。

1. GitHubでリポジトリのページを開く（`https://github.com/ユーザー名/リポジトリ名`）
2. 上部タブの **「Settings」** をクリック
3. 左メニューの **「Pages」** をクリック
4. **Source**: 「Deploy from a branch」を選択
5. **Branch**: 「main」 / 「/ (root)」を選択して **「Save」**
6. 数分待つと、以下のURLでサイトが公開される

```
https://ユーザー名.github.io/リポジトリ名/
```

---

## 補足: ソースコードとサーバーの仕組み

今回の構成では、**ソースコードもサーバーもすべてGitHub**にあります。
AWSやレンタルサーバーなどは一切使っていません。

```
GitHub（1つで2役）
├── リポジトリ = ソースコード置き場（HTML/CSS/JSファイル）
└── GitHub Pages = Webサーバー（URLでサイトを公開）
```

つまり、GitHubのアカウント1つで「コードの管理」と「サイトの公開」の両方ができます。

## よくある質問

### Q: 料金はかかりますか？
- **GitHub**: 無料（Publicリポジトリ + GitHub Pages）
- **Claude**: Claudeのサブスクリプション料金のみ（Pro / Team / Enterprise）

### Q: プログラミングの知識は必要ですか？
不要です。「こういう画面を作って」と日本語で伝えるだけで、Claude Codeがコードを書きます。ただし、HTMLやCSSの基本を知っていると、より的確な指示が出せます。

### Q: 作ったサイトを後から修正できますか？
はい。同じリポジトリでClaude Codeのセッションを開き、「ここを変えて」と伝えれば修正してくれます。

### Q: 会話の内容は次のセッションに引き継がれますか？
会話内容はリセットされます。ただし、リポジトリに `CLAUDE.md` というファイルを置いておくと、新しいセッション開始時にClaude Codeが自動的に読み込み、プロジェクトの背景を把握した状態で始められます。

### Q: 変更がブラウザに反映されないのですが？
ブラウザのキャッシュが原因です。**Ctrl + Shift + R**（強制リロード）を試してください。

### Q: 会社のネットワークからサイトが見れません
企業のファイアウォールが原因の可能性があります。スマホのテザリングなど、社外ネットワークからアクセスしてみてください。独自ドメインを使う場合は `.com` や `.jp` を選ぶと企業ネットワークでブロックされにくいです。
